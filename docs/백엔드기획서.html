01 · Architecture
Supabase 아키텍처 개요
BaaS(Backend as a Service) — 서버 코드 없이 DB·Auth·API 일괄 해결
핵심 선택 이유
서버를 직접 만들지 않고도
PostgreSQL + 인증 + REST API를 동시에 얻는다
🗄 PostgreSQL DB
  • 관계형 데이터 구조
  • 감정-말씀 데이터에 최적
  • 복잡 쿼리 지원 (통계)
  • 자동 REST API 생성
🔐 Supabase Auth
  • Google OAuth 2.0 내장
  • JWT 토큰 자동 관리
  • 세션 자동 갱신
  • 별도 인증 서버 불필요
🛡 Row-Level Security
  • DB 레벨 접근 제어
  • auth.uid() = user_id
  • anon key 공개 가능
  • 서버 코드 없이 보안
02 · Init Code
클라이언트 초기화
index.html에 CDN으로 SDK 로드 → createClient로 연결
Supabase 클라이언트 초기화 (index.html)
// CDN으로 SDK 로드 (head 태그)
// <script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2/dist/umd/supabase.min.js"></script>

const SUPABASE_URL = 'https://pzakpaomnmahcugkzdff.supabase.co';
const SUPABASE_KEY = 'sb_publishable_...'; // anon key (RLS로 보호됨)
let sb = null;   // Supabase 클라이언트
let sbUser = null; // 현재 로그인 사용자

function initSupabase() {
  if (location.protocol === 'file:' || typeof supabase === 'undefined') return;
  sb = supabase.createClient(SUPABASE_URL, SUPABASE_KEY);

  // 로그인 상태 변화 감지 (핵심!)
  sb.auth.onAuthStateChange((event, session) => {
    sbUser = session?.user || null;
    updateAuthUI();
    if (event === 'SIGNED_IN') migrateLocalToServer();
  });
}
03 · Auth Flow
Google OAuth 인증 플로우
1
로그인 버튼 클릭
설정 오버레이 > 계정 섹션 > "Google로 로그인" 버튼 → loginWithGoogle() 호출

Supabase OAuth 리디렉션

Google 인증 완료
사용자가 Google 계정 선택 → Supabase callback URL로 리디렉션

04 · Security
Row-Level Security (RLS)
DB 레벨에서 "본인 데이터만 접근 허용" — 서버 코드 없이 완전한 보안
-- 모든 테이블에 RLS 활성화
ALTER TABLE profiles       ENABLE ROW LEVEL SECURITY;
ALTER TABLE visit_dates    ENABLE ROW LEVEL SECURITY;
ALTER TABLE mood_history   ENABLE ROW LEVEL SECURITY;
ALTER TABLE saved_verses   ENABLE ROW LEVEL SECURITY;

-- 정책: 본인 데이터만 CRUD 허용
CREATE POLICY "본인만 접근" ON profiles
  FOR ALL USING (auth.uid() = id);

CREATE POLICY "본인만 접근" ON mood_history
  FOR ALL USING (auth.uid() = user_id);

-- 결과: anon key가 외부에 노출돼도 타 사용자 데이터 열람 불가
-- auth.uid() = 현재 JWT 토큰의 사용자 ID
05 · Data Sync
데이터 동기화 전략
localStorage 우선 → 로그인 시 Supabase 동기화 (하위 호환 유지)
마이그레이션 전략 (최초 로그인 시)
async function migrateLocalToServer() {
  if (!sbUser || !sb) return;

  // 서버에 이미 데이터가 있으면 → 서버 우선
  const { data } = await sb.from('mood_history')
                            .select('id').limit(1);
  if (data?.length > 0) {
    toast('클라우드 데이터를 불러왔어요 ☁️'); return;
  }

  // 서버 비어있으면 → localStorage 데이터 일괄 업로드
  const moods = lsGet('moodHistory') || [];
  if (moods.length) {
    await sb.from('mood_history').insert(
      moods.map(m => ({ user_id: sbUser.id, date: m.date,
        emotion: m.emotion, topic: m.topic, need: m.need,
        diary_text: m.diary || null }))
    );
  }
  toast('기존 데이터를 클라우드에 저장했어요 ☁️');
}
06 · CRUD Pattern
실시간 동기화 패턴
데이터 저장 시 동기화 (saveMood 예시)
function saveMood() {
  const { emotion, topic, need } = sel;
  const today = todayStr();

  // 1. localStorage 먼저 저장 (오프라인 대응)
  const hist = lsGet('moodHistory') || [];
  hist.unshift({ date: today, emotion, topic, need });
  lsSet('moodHistory', hist.slice(0, 30));

  // 2. 로그인 상태면 Supabase에도 비동기 동기화
  if (sbUser && sb) {
    sb.from('mood_history')
      .insert({ user_id: sbUser.id, date: today, emotion, topic, need })
      .then();   // 실패해도 UI 블록 안 됨
  }
}
