08 · Security
보안 고려사항
데이터 보호 및 키 관리 체계
✓RLS (Row-Level Security) — 모든 테이블에 user_id = auth.uid() 정책 적용. anon key가 유출돼도 타 사용자 데이터 접근 불가
✓Google OAuth — 비밀번호 저장 없음. 구글이 인증 책임. JWT 토큰 자동 갱신
✓anon key 공개 — RLS 보호 하에 프론트에 노출 가능 (설계상 안전)
⚠service_role key — 절대 코드/GitHub에 노출 금지. 서버 전용
⚠OAuth redirect URI — Google Cloud Console + Supabase 양쪽에 정확히 등록 필요
✓.gitignore — .env, service_role key, 개인 메모 파일 깃 추적 제외
