02 · Core Functions
핵심 함수 명세
init()
앱 진입점. DOM 로드 후 실행. 별·테마·언어·스트릭·Supabase 초기화 순서로 호출.
초기화Supabase
initSupabase()
Supabase 클라이언트 생성 + onAuthStateChange 등록. file:// 프로토콜에서는 실행 안 됨.
Supabase
migrateLocalToServer()
로그인 직후 1회 실행. 서버 데이터 없으면 localStorage → Supabase 일괄 업로드. async/await.
SupabaselocalStorage
buildResult()
4단계 완료 후 말씀·묵상·기도문·격려문을 조합해 결과 화면 렌더링. getVerseForMood() 호출.
UI
saveMood() / toggleSave() / saveDiary()
각각 감정기록·말씀저장·일기 저장. localStorage 저장 후 sbUser 존재 시 Supabase에도 비동기 동기화.
localStorageSupabase 동기화
updateStreak() / calcStreak()
오늘 방문 기록 후 연속 스트릭 계산. 60일 이전 데이터 자동 정리. visit_dates에도 동기화.
localStorageSupabase
lsGet(k) / lsSet(k, v)
localStorage 래퍼. JSON 자동 파싱/직렬화. try/catch로 예외 처리. 앱 전체에서 사용하는 핵심 유틸.

인증 상태 감지
onAuthStateChange
sb.auth.onAuthStateChange(
  (event, session) => {
    sbUser = session?.user || null;
    updateAuthUI();
    if (event === 'SIGNED_IN')
      migrateLocalToServer();
  }
);
테이블별 CRUD 패턴
// mood_history — 감정 기록 추가
await sb.from('mood_history').insert({
  user_id: sbUser.id, date: todayStr(),
  emotion, topic, need
});

// mood_history — 일기 업데이트
await sb.from('mood_history').update({ diary_text: text })
  .eq('user_id', sbUser.id).eq('date', todayStr());

// visit_dates — 중복 방지 upsert
await sb.from('visit_dates').upsert(
  { user_id: sbUser.id, visit_date: today },
  { onConflict: 'user_id,visit_date' }
);

// ★ v3.0 감정 통계 쿼리 (예정)
const { data } = await sb
  .from('mood_history')
  .select('emotion, date')
  .eq('user_id', sbUser.id)
  .gte('date', weekAgo)
  .order('date');
Service Worker
SW 캐시 전략
캐시 전략 (god-app-v6)
  • index.html: Network-first (즉시 업데이트)
  • 정적 에셋: Cache-first (sw.js, icon.svg)
  • 업데이트: SKIP_WAITING 메시지 패턴
  • 새 버전: update-banner 표시 → 사용자 확인
업데이트 흐름
  • Vercel 배포 → 새 index.html 서버 전송
  • SW updatefound 이벤트 감지
  • 새 SW 설치 완료 시 배너 표시
  • 사용자 클릭 → SKIP_WAITING → 새로고침

05 · Deploy & Maintenance

배포 파이프라인
1
코드 수정 + 검증
index.html 편집 후 node 검증 스크립트 실행
node -e "..."
2
git commit
feat/fix/docs 프리픽스 + 버전 명시
git commit -m "feat: v2.X ..."
3
GitHub push
main 브랜치 push → Vercel 자동 배포 트리거
git push ... main
4
Vercel 자동 배포
~30초 내 전 세계 CDN 배포 완료
5
SW 업데이트 배너
기존 사용자에게 업데이트 배너 자동 표시
유지보수 핵심 주의사항
  • 단일 파일 원칙: 새 JS/CSS 파일 추가 시 SW 캐시 등록 필요
  • div 균형: HTML 추가 시 항상 open/close 쌍 확인
  • LANG_DATA: 텍스트 추가 시 ko/en/ja 3개 동시 추가
  • SW 캐시명: 정적 에셋 추가 시 god-app-vN 버전 올릴 것
  • Supabase 신규 테이블: 반드시 RLS 활성화 + 정책 추가
  • 버전 업: APP_VERSION 상수 + CHANGELOG.md 동시 업데이트
추천 개발 도구
  • 에디터: VS Code + Prettier
  • 검증: node -e "..." (브라우저 없이 문법 검사)
  • DB 확인: Supabase Dashboard → Table Editor
  • 배포 확인: Vercel Dashboard → Deployments
