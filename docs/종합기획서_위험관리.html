05 · Risk Response
위험별 대응 전략
식별된 7개 리스크 · 우선순위별 대응 방안
HIGH
보안 사고 — 사용자 데이터 유출
RLS 설정 누락 또는 service_role key 노출 시 전체 DB 접근 가능
대응: RLS 전 테이블 적용 완료 · .gitignore로 key 파일 제외 · 정기 보안 감사
HIGH
콘텐츠 반복 — 말씀 Pool 소진
144개 말씀이 반복 노출되면 사용자 이탈 가속 (특히 매일 사용자)
대응: v3.0까지 300개 이상으로 확장 · AI 생성 말씀 보조 검토 · 계절별 큐레이션
MED
Supabase DB 한도 초과 (500MB)
사용자 증가 시 무료 플랜 한도 도달 → 서비스 중단 또는 비용 발생
대응: 90일 이상 mood_history 아카이빙 정책 · 조기에 Supabase Pro 전환 ($25/월) 수익화 선행 필요
MED
Google OAuth 정책 변경
Google이 OAuth 정책 변경 시 로그인 기능 중단 가능
대응: Apple OAuth 또는 카카오 OAuth 추가 준비 (대체 인증 수단) · 이메일 Magic Link 보조 옵션
MED
결제 시스템 미구축 → 수익 지연
Premium 기능이 있어도 결제 수단이 없으면 수익화 불가
대응: v3.0에서 토스페이먼츠 또는 Stripe 연동 · 후원 모델 선행으로 소규모 수익 확보
LOW
1인 개발 번아웃 / 업데이트 중단
개발자 부재 시 버그 수정 · 기능 업데이트 중단
대응: 코드 문서화 (기술명세서 유지) · CHANGELOG 관리 · 핵심 기능은 단순 유지
LOW
대형 앱과의 경쟁 (YouVersion 등)
리소스 많은 경쟁자가 감정 기반 기능 복제 가능
대응: 감정 통계 · AI 개인화로 빠른 차별화 · 한국 특화 (한국어 말씀 품질) · 커뮤니티 록인

06 · Contingency
컨틴전시 플랜
최악의 시나리오 대비 즉각 대응 방안
🚨 서비스 중단 시

상황: Vercel 배포 실패 또는 Supabase 장애
즉각 대응:
· GitHub Pages 백업 URL로 즉시 전환
· SW 캐시로 오프라인 사용 유지
· Supabase 장애 시 localStorage 자동 폴백
· 복구 예상 시간 사용자 공지 (SNS)

⚠️ 데이터 손실 시

상황: DB 삭제 또는 마이그레이션 오류
즉각 대응:
· Supabase 자동 백업 복원 (최대 7일 이전)
· 사용자 localStorage 데이터 유지 (이중 저장)
· 피해 사용자 Premium 무료 보상
· 재발 방지: 주간 백업 스크립트 자동화

💸 인프라 비용 급증 시

상황: 사용자 급증으로 Supabase 무료 한도 초과
즉각 대응:
· Supabase Pro 전환 ($25/월)
· 데이터 보존 정책 강화 (90일 자동 삭제)
· 선결제 후원 캠페인으로 비용 확보
· 장기: 자체 PostgreSQL 서버 이전 검토

🚀 급격한 성장 시

상황: 바이럴로 MAU 10,000 돌파
즉각 대응:
· Supabase Pro 즉시 전환
· Vercel Pro로 업그레이드
· Premium 플랜 조기 출시
· 콘텐츠 업데이트 속도 긴급 강화
